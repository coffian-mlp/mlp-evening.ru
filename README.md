# 🦄 MLP-Evening: Поняшный Вечерок

**MLP-Evening** — это уютная и технологичная платформа для совместных киносеансов, созданная с любовью к дружбе и магии кода. Изначально проект задумывался для ламповых просмотров *My Little Pony: Friendship is Magic*, но его гибкость позволяет наслаждаться любым контентом в хорошей компании.

Ключевая особенность проекта — гибкая архитектура чата. Мы реализовали **гибридную систему**:
*   **Centrifugo (Go):** Мощный WebSocket сервер для масштабирования (тысячи онлайн).
*   **SSE (PHP):** Встроенный режим (Server-Sent Events) для работы "из коробки" без дополнительных зависимостей (для простых хостингов).

В комплекте идет умный генератор плейлистов, учитывающий историю просмотров, система ролей и удобная панель управления для Принцесс (администраторов).

## 🌟 Возможности

### 📺 Плеер и Просмотр
*   **Универсальный Плеер:** Поддержка вставки стримов через iframe (Twitch, GoodGame, YouTube и др.).
*   **Умный Плейлист:** Алгоритм генерации расписания на основе весов популярности и истории просмотров.
*   **Двусерийники:** Система знает, что некоторые эпизоды — это единое целое, и всегда ставит их парой!
*   **📅 Великое Расписание:** Встроенный календарь (`/schedule.php`) с поддержкой повторяющихся событий, умным таймером и автоматической генерацией плейлистов к началу стрима!

### 🤖 Лира-Бот (AI Ассистент)
*   **Умные ответы:** Интеграция с LLM (YandexGPT/OpenAI) для общения с пользователями в чате.
*   **Контекст и Память:** Бот помнит историю сообщений и отвечает с учетом контекста беседы.
*   **Спонтанность:** Бот умеет сам инициировать диалог, прерывая неловкое молчание, но не разговаривает сам с собой.
*   **Интеграция с расписанием:** По команде `/расписание` или `/schedule` бот изучает календарь и выдает анонс ближайшего события, а также сам делает авто-анонсы за 60 и 15 минут до старта!
*   **Гибкие настройки:** Управление промптом, токенами и поведением бота прямо из панели администратора.

### 💬 Магический Чат (Zero-Dependency Realtime)
*   **Мгновенная доставка:** Технология SSE обеспечивает живое общение без задержек.
*   **Markdown и Форматирование:**
    *   Жирный, *курсив*, ~~зачеркнутый~~, `код` и `> цитаты`.
    *   Спойлеры `||скрытый текст||` для защиты от лишней информации.
*   **Медиа-магия (Embeds):** Автоматическое превращение ссылок в картинки и видео. Загрузка файлов Drag & Drop и вставка из буфера обмена (Paste).
*   **Удобство общения:**
    *   **Упоминания:** Нажми на имя пони или напиши `@Ник`, чтобы позвать друга.
    *   **Цитирование:** Красивые карточки цитат с переходом к оригиналу.
    *   **Редактирование:** Возможность исправить опечатку или удалить сообщение (Soft Delete).
    *   **Реакции:**
        *   Выражай эмоции (Лайк, Смех, Слезы и др.) одной кнопкой.
        *   **Smart Tooltip:** Наведи на реакцию, чтобы увидеть список пони, которые её поставили (с их аватарками!).
        *   Мгновенное обновление счетчиков у всех пользователей.

### 🦄 Режимы работы чата

Проект поддерживает два драйвера для работы чата, переключаемых в `config.php`:

1.  **Centrifugo (Рекомендуется)** 🚀
    *   Использует WebSocket для мгновенной доставки сообщений.
    *   Выдерживает тысячи одновременных подключений.
    *   Снимает нагрузку с PHP и базы данных.
    *   **Инструкция по установке:** [DEPLOY_CENTRIFUGO.md](DEPLOY_CENTRIFUGO.md)

2.  **SSE (Server-Sent Events)** 🐢
    *   Работает "из коробки" на голом PHP без дополнительного ПО.
    *   Использует постоянные HTTP-соединения.
    *   **⚠️ Внимание:** Этот режим может работать нестабильно на слабых серверах или виртуальном хостинге (возможны задержки сообщений из-за буферизации Nginx/Apache, блокировок сессий или таймаутов).
    *   Мы добавили этот режим для совместимости, но рекомендуем использовать Centrifugo для стабильной работы.

### 🛡️ Модерация и Администрирование
*   **Иерархия:** Система ролей (Пользователь, Модератор, Администратор).
*   **Безопасность:**
    *   Восстановление пароля через Email (SMTP) с генерацией безопасных токенов.
    *   **Защита от подбора паролей (Brute-force):** Умная блокировка по IP. При подозрительной активности система предлагает пройти "Испытание Гармонии" (капчу), а при настойчивых атаках — временно блокирует доступ.
*   **Инструменты Порядка:** Бан (Ban), Мут (Mute - временная тишина), Очистка (Purge).
*   **Контекстное Меню:** Быстрые действия по правому клику на сообщении.
*   **Всевидящее Око (Audit Log):** Подробный журнал действий модераторов.
*   **Debug Mode:** Включение отображения ошибок PHP (`display_errors`) прямо из админки для быстрой отладки на лету.
*   **Дашборд:** Управление пользователями, плейлистами и настроек сайта в одном месте.

## 🛠️ Технологии

*   **Backend:** PHP 8.3 (Extensions: `curl`, `opcache`) — чистая магия, никаких фреймворков.
*   **Frontend:** HTML5, CSS3 (Flexbox/Grid), jQuery (локальный) — легкость и скорость.
*   **Database:** MySQL 5.7 — надежное хранилище знаний.
*   **Container:** Docker & Docker Compose — развертывание одной командой.

## 🧠 Архитектура Ядра (src/)

Сердце проекта — набор PHP классов, реализующих всю бизнес-логику без использования сторонних фреймворков.

### 🧩 Компоненты (src/Components/)

Мы используем модульную архитектуру, вдохновленную Bitrix. Каждый функциональный блок — это независимый компонент со своей логикой, шаблонами и стилями.

*   **`Auth`**: Модальные окна входа, регистрации и восстановления пароля.
*   **`Chat`**: Основной компонент чата. Поддерживает режимы `embedded` (встроенный) и `popup` (отдельное окно).
*   **`DbAdmin`**: Мощный инструмент для управления базой данных прямо из админки (CRUD, фильтрация, экспорт CSV).
*   **`Profile`**: Модальное окно настроек пользователя (аватар, цвет ника, смена пароля).
*   **`VideoPlayer`**: Обертка для плеера стрима.
*   **`Admin*`**: Семейство компонентов панели управления (`AdminUsers`, `AdminPlaylist`, `AdminHistory` и др.).

### 🏛️ Основные Классы (src/Core/ & src/)

### `Core` (src/Core/)
Фундамент модульной системы.
*   `Application`: Главный класс-распорядитель (Singleton). Управляет заголовками, подключением CSS/JS и компонентов.
*   `Component`: Базовый абстрактный класс для всех компонентов. Обеспечивает логику подключения шаблонов.

### `Database`
Обертка над MySQLi, реализующая паттерн **Singleton**. Гарантирует единое подключение к базе данных на протяжении всего скрипта.
*   `getInstance()`: Получение экземпляра.
*   `getConnection()`: Прямой доступ к объекту `mysqli`.

### `Auth`
Статический класс для управления сессиями и безопасностью.
*   `check()` / `requireLogin()`: Проверка авторизации.
*   `isAdmin()` / `isModerator()`: Проверка прав доступа.
*   `generateCsrfToken()` / `checkCsrfToken()`: Защита форм от CSRF-атак.

### `ConfigManager`
Управление глобальными настройками сайта (хранятся в `site_options`).
*   `getInstance()`: Получение экземпляра (Singleton).
*   `getOption($key, $default)`: Получение значения настройки.
*   `setOption($key, $value)`: Сохранение настройки.

### `OnlineManager`
Управление статусом онлайн и подсчетом гостей.
*   `beat($sessionId, $userId, ...)`: Обновление времени последней активности (Heartbeat).
*   `getOnlineStats($window)`: Получение списка пользователей и количества гостей за последние N минут.
*   `cleanup($minutes)`: Очистка старых сессий из базы.

### `CaptchaManager`
Управление игровым процессом "Испытание Гармонии" (капчей).
*   `start()`: Инициализация сессии капчи (выбор случайного пони).
*   `checkAnswer($answer)`: Проверка ответа и переход к следующему этапу (Текст -> Картинка -> Спайк).

### `CentrifugoService`
Обертка для работы с HTTP API сервера Centrifugo.
*   `publish($channel, $data)`: Отправка сообщения в канал.
*   `broadcast($channels, $data)`: Массовая рассылка.
*   `generateToken($userId)`: Генерация JWT токена для клиента.

### `Mailer` & `SimpleSMTP`
Отправка транзакционных писем (например, сброс пароля).
*   `Mailer::sendPasswordReset($email, $link)`: Формирование и отправка письма.
*   `SimpleSMTP`: Легковесная реализация SMTP клиента на сокетах (без PHPMailer).

### `UserManager`
Управление пользователями, их профилями и настройками.
*   `getAllUsers()`: Получение списка пользователей с `JOIN`-ом их настроек (цветов, аватарок) из таблицы `user_options`.
*   `updateUser($id, $data)`: Умное обновление — автоматически распределяет данные между таблицей `users` и `user_options`.
*   `banUser()` / `muteUser()` / `logAction()`: Логика наказаний и ведение журнала действий (Audit Log).

### `ChatManager`
Логика чата, обработка сообщений и Markdown.
*   `getMessages($limit)`: Выборка сообщений для SSE потока (с обработкой удаленных и отредактированных).
*   `addMessage()`: Добавление сообщения с проверкой статуса пользователя (не забанен ли, нет ли мута) и анти-спам таймером.
*   `purgeMessages()`: Массовое удаление (Soft Delete) сообщений нарушителя.

### `EpisodeManager`
Магия составления расписания и управления контентом.
*   `getEveningPlaylist()`: Алгоритм генерации плейлиста. Учитывает веса эпизодов, историю просмотров и связки двусерийников.
*   `voteForEpisode()`: Учет голосов "Хочу посмотреть".
*   **Кеширование:** Встроенная система файлового кеширования (File-Based Cache) для списка эпизодов.
    *   `getAllEpisodes()`: Кеширует результат выборки в JSON файл на 60 секунд.
    *   При любых изменениях (голосование, обновление плейлиста) кеш мгновенно сбрасывается для поддержания актуальности данных.

### `StickerManager`
Управление коллекцией стикеров.
*   `getAllPacks()`: Список коллекций (паков).
*   `createPack()` / `updatePack()` / `deletePack()`: Управление паками (с автоматическим удалением файлов).
*   `addSticker()`: Добавление стикера.
*   `importFromZip()`: Магия массовой загрузки из архива с автоматической транслитерацией и генерацией кодов.

### `UploadManager`
Обработка загружаемых файлов.
*   `uploadFromPost()`: Валидация и сохранение файлов из форм.
*   `uploadFromUrl()`: Скачивание изображений по ссылке (для аватарок).

### `SocialAuthService` (src/Social/)
Единая точка входа для авторизации через социальные сети.
*   `handleLogin()`: Умная логика входа:
    1.  Если аккаунт привязан -> Вход.
    2.  Если пользователь уже вошел -> Привязка соцсети к профилю.
    3.  Если новый пони -> Автоматическая регистрация и вход.
*   `TelegramProvider`: Реализация проверки подписи виджета Telegram (Hash Validation).

## 🚀 Как запустить (Локально)

1.  Убедитесь, что у вас установлен **Docker** и **Docker Compose**.
2.  Клонируйте этот репозиторий.
3.  Создайте конфигурационные файлы:
    ```bash
    cp config.sample.php config.php
    cp centrifugo_config.sample.json centrifugo_config.json
    ```
4.  Запустите магию одной командой:
    ```bash
    docker-compose up -d --build
    ```
4.  Сайт откроет свои двери по адресу: `http://localhost:8080`

### 🔑 Доступы
*   **MySQL:** Конфигурация в `docker-compose.yml` и `config.php`.
*   **Сайт:** Регистрация открыта для всех пони.
*   **Админка:** Доступна пользователям с ролью `admin` (кнопка 🔧 в меню).
